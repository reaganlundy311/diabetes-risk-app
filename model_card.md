## 1. Model details

- **Owner.** SciEncephalon AI Summer Intern Series 2026 — Project 04.
- **Author.** Reagan Lundy.
- **Architecture.** `sklearn.ensemble.HistGradientBoostingClassifier`, wrapped in
  `sklearn.calibration.CalibratedClassifierCV` with isotonic regression
  (`cv=3` or `cv=5`).
- **Inputs (8 non-invasive tabular features).** `pregnancies`, `glucose`,
  `blood_pressure` (diastolic), `skin_thickness`, `insulin`, `bmi`, `dpf`
  (diabetes pedigree function — a family-history score), `age`.
- **Output.** Calibrated probability `P(diabetes=1)` in `[0, 1]`.
- **Reproducibility.** `np.random.seed(42)`, `random_state=42` everywhere.

## 2. Intended use

- **In scope.** Classroom demonstration of (a) calibrated probability
  estimation, (b) sensitivity / specificity threshold sweeps, (c) lift/gain
  targeting analysis, (d) subgroup performance review, and (e) the separation
  between a prediction layer and a live-or-template messaging layer.
- **Out of scope.** Clinical decision support, screening at the point of care,
  insurance underwriting, employment screening, or any use that affects a
  real person.

## 3. Training data

- **Default (notebook).** Synthetic dataset generated by
  `data/loader.py::load_synthetic` (500–768 rows, schema mirrors PIMA exactly).
- **Stretch / Week 2.** Real PIMA Indians Diabetes dataset
  (`https://raw.githubusercontent.com/jbrownlee/Datasets/master/pima-indians-diabetes.data.csv`,
  OpenML mirror `https://www.openml.org/d/37`, 768 rows).
- **Cohort.** The real PIMA dataset contains **only Pima women aged 21+**. It is
  not representative of men, of non-Pima populations, or of younger / older
  patients.

## 4. Metrics (synthetic baseline)

Reported on the held-out 25% test split with the default 0.50 threshold.

| Metric | Value (synthetic) | What it means |
|---|---|---|
| AUC | ~0.85 | Discrimination — how well the model separates positives from negatives. |
| Accuracy | ~0.78 | Fraction correct overall (less useful with class imbalance). |
| Sensitivity (recall) | ~0.70 | Of true positives, how many we catch. |
| Specificity | ~0.82 | Of true negatives, how many we leave alone. |
| Brier score | ~0.15 | Probability-calibration quality (lower is better). |

## 5. Known limitations & biases

- **Cohort bias.** PIMA is exclusively Pima women aged 21+. The model has NOT
  been validated on other populations.
- **Missing-encoded-as-zero.** PIMA encodes missing `glucose`, `blood_pressure`,
  `skin_thickness`, `insulin`, and `bmi` as `0`. Naive training preserves those
  zeros as if they were real measurements. Decide and document your
  imputation strategy.
- **Class imbalance.** ~35% positive class. AUC is robust to this, but the
  default 0.50 threshold may not match your deployment context.
- **Non-invasive inputs only.** True diagnosis requires A1C or fasting plasma
  glucose tests — this model does not replace them.
- **Public app.** `https://diabetes-risk-reagan-lundy.streamlit.app` hosts the
  Streamlit demo.
- **Coach layer.** `src/lifestyle_coach.py::llm_compose` supports a live OpenAI
  coach when `OPENAI_API_KEY` is configured, with a safe template fallback when
  no key is available. The layer must NEVER prescribe, dose, or name specific
  drugs.

## 6. Ethical considerations

- Every user-facing output (notebook, Streamlit app, coach message) includes
  **"not medical advice"** verbatim.
- No PHI is collected; the Streamlit demo holds inputs in browser session
  state only.
- The fairness layer (`src/fairness.py`) reports per-subgroup AUC, sensitivity,
  and specificity by BMI / age bucket. A meaningful gap should be disclosed in
  this card before any external demo.

## 7. Caveat that always applies

This is a learning artifact. If you, the reader, are worried about your own
diabetes risk, please see a primary-care provider. They can order an A1C test
that takes minutes and reflects three months of blood sugar — a real screen,
not a probability from a high-school project.
